Order status

created

The order was created using the booking system but it was not delegated neither paid.

ordered

The order was created using the booking system and it was delegated or paid. Also, the order may had been created by an admin using the BO.

prepaid

The order was paid but no delivered. The previous status may had been created or ordered.

delivered

The order is delivered (done and pdf uploaded) but not paid. The previous status was ordered.

completed

The order is done with pdf uploaded and paid. The previous status was delivered or prepaid.

Notifications

A notification is the representation an email that may had been sended or not to an email address.
Is created in the moment that the server decide to send an email.
It has a flag that indicate if the email has been sended or not but this only indicate that the order has been sended to the smtp server by the diags server. The real email may had been delayed depends on the smtp overload.

An User (Admin, Diag or Client), may had N notifications associated at any time. The

Next, a list of available notifications.

  • ADMIN_NEW_ACCOUNT

    When this type of user is created, this notification is generated to the user email.

  • DIAG_NEW_ACCOUNT

    When this type of user is created, this notification is generated to the user email.

  • CLIENT_NEW_ACCOUNT

    When this type of user is created, this notification is generated to the user email.

  • NEW_CONTACT_FORM_MESSAGE

    Its fired when someone use the contact form of the web site.

  • DIPLOME_EXPIRATION

    Its fired one time in an automated-task when a diplome is expired.

  • ORDER_CREATED

    Its fired for the Client and Diag man once an order has been created. Status created.

  • PASSWORD_RESET

    Its fired for user after he request a password reset.

  • PAYMENT_LINK

    Its fired for client of an order ('to' targeting the landlord email) after he request to delegate an order payment.

  • ORDER_PAYMENT_SUCCESS

    When an order had been paid succefully. Targets the Client, the diag, and all the Admins.
